## Context

Assignment is the step that converts the triage decision into an operational commitment. The case coordinator confirms the package code, the delivery mode and the target dispatch time with ADAC, and the physician-nurse team begins movement. Because pricing is fixed at the package level, pre-authorization is a single yes/no decision against the published catalogue rather than a line-item negotiation.

## Key points

- **One package code, one decision.** The case is mapped to a single code from the 65-package catalogue, taken directly from the 9-category framework.
- **Mode confirmed and locked.** Clinic, hotel room or mobile unit is set in writing, so the operational plan matches the clinical plan.
- **Pre-authorization is binary.** Yes/no on the published package — no itemized negotiation, no surprises later in the invoice.
- **Dispatch target communicated.** Both ADAC and the AG Holder receive the dispatch time so expectations are aligned at both ends.
- **Equipment and team match the package.** The physician, nurse and kit are loaded against the package profile, not the worst-case scenario.

## What this means for ADAC

Assignment is where the package-based pricing model earns its keep. One decision, one number, one written commitment — which means a fast yes from ADAC, a fast start on the HMC side and a paper trail that lines up cleanly with the invoice. The claims cycle starts here because the cost basis is fixed here.
